# Skill Agent 文档总览

本文档集描述的是 2026-07-16 仓库中的实际实现，而不是历史设计稿。

## 项目定位

Skill Agent 是一个以 `Skill` 为中心的多 Agent 编排项目。

- `ManagerAgent` 负责意图识别、技能选择、工具规划
- `LearningAgent` 负责执行工具任务和 DAG
- `OrchestratorAgent` 负责整合结果并生成最终回复
- `SkillTrainer` 负责把用户显式教学沉淀为技能文件

系统同时提供三种入口：

- FastAPI REST + WebSocket PubSub
- Vue 3 前端
- CLI / Streamlit 同步入口

## 当前实现状态

- 主执行入口已经统一到异步 `Agent.handle()`
- `Agent.chat()` 作为 CLI / Streamlit 的同步适配层保留
- 后端 WebSocket 路径是 `/pubsub`，不是旧版 `/ws/chat`
- 会话按 `client_id` 复用，带 TTL GC
- 工具统一由 `ToolHub` 管理，支持启动连接和关闭释放资源
- `weather_query` 与 `web_search` 已改为异步实现
- 自演化面板已接入真实后端 API
- 补丁审批现在会把技能修改落到 YAML 文件

测试基线：

- `pytest -q` 通过
- 当前已知保留 1 个 `skip`，是 `TestClient + PubSub` 的时序不稳定用例

## 当前已知限制

- `backend/websocket_handler.py` 与部分前端源码仍有编码问题，影响可读性但不影响现有测试
- `SelfReflectLoop`、`ExecutionCritic` 的实现已落地，但策略仍偏规则化
- WebSocket 端到端自动化测试仍不稳定
- 前端缺少系统化自动化测试
