> 本目录是对项目 **现状分析 + 架构改进设计** 的全套文档,适用于二次开发、新成员上手、阶段性评审。

---

## 0. 项目定位

**Skill Agent** 是一个**以"技能(Skill)"为一等公民的 LLM 智能体框架**。

核心理念:**技能 = 方法论 + 步骤 + 可选代码**。

通过"技能驱动 + 工具执行 + 方法论整合"的方式,让 LLM 能:
1. 听懂用户意图
2. 选用合适的方法论(技能)指导执行
3. 调用工具获取数据
4. 按方法论生成回答
5. **接受用户的"教导"并沉淀为新技能**

### 2.2 已实现但有缺陷

- Manager 的 LLM 输出**JSON 解析靠正则,无重试**
- Learning 创建了 `ThreadPoolExecutor` 但**任务始终串行**
- Orchestrator 的"按方法论生成回答"**只是 prompt 拼接,无强制约束**
- 技能加载支持 `code` 字段但**运行时从不读取**
- WebSocket 日志订阅在 `try` 加但**`finally` 才 unsubscribe,长时间运行会内存泄漏**

### 2.3 未实现(关键缺失)

- **用户"教导技能"闭环**(README/ARCHITECTURE 反复承诺,代码无对应)
- **多轮对话上下文**(core/agent.py 的 `self.context` 从未使用)
- **工具调用依赖图 / 重试 / 缓存**
- **技能版本管理 / 冲突仲裁 / 失效检测**
- **统一的 Agent 基类**(Manager/Orchestrator 都自管 LLM 调用)
- **结构化输出校验**(JSON Schema / retry)
- **前端流转可视化**(只显示最终文本,看不到 Manager/工具/方法论步骤)
- **`.env.example`**(README 提及但文件不存在)
- **测试 / CI / Docker**

## 3. 改进目标

### 3.1 短期(1~2 周)

1. **统一 `BaseAgent`** 抽象,固化 LLM 调用、trace 关联、错误处理
2. **Manager 输出 JSON Schema 化**,失败时**自动重试 + 降级**
3. **Orchestrator 真正按技能步骤执行**,LLM 只负责填值
4. **WebSocket 协议单一化**,日志订阅确保 `finally` 清理
5. 补 `.env.example`、Docker、基础测试

### 3.2 中期(1~2 月)

1. **技能转 DAG**:每个技能步骤声明输入/输出,执行器按拓扑序调度
2. **工具调用支持依赖关系**:Manager 表达"先查天气再决定搜什么"
3. **教导闭环**:识别"教导意图" → 解析 method/steps → 写入技能库
4. **多轮对话 Context 持久化**(SQLite/Redis)
5. **前端重做,把"流转可视化"作为主界面**

### 3.3 长期

1. 技能版本化 + 评估机制
2. 工具结果缓存与去重
3. 任务失败的反思/重规划回路
4. 多 LLM 提供商适配(当前 OpenAI SDK 兼容,但未抽象)

## 4. 架构总览图(目标态)

```
                   ┌─────────────────────────────────────┐
                   │  Skill Registry (Versioned / DAG)   │
                   └────────────────┬────────────────────┘
                                    │ semantic match
                                    ▼
 ┌──────────┐   user msg   ┌────────────────┐    DAG plan   ┌─────────────┐
 │  User    │─────────────▶│ Manager Agent  │──────────────▶│   Learning  │
 └──────────┘              │  (意图+规划)    │               │    Agent    │
       ▲                   └────────┬───────┘               │  (执行工具) │
       │                            │ structured             └──────┬──────┘
       │                            ▼                               │
       │                    ┌────────────────┐                      │
       │                    │   Skill DAG    │  ◀──── DAG plan      │
       │                    │   Executor     │                      │
       │                    └────────┬───────┘                      │
       │                             │ per-step output              │
       │                             ▼                              ▼
       │                     ┌──────────────────────────────────────┐
       └─────  streaming  ───│   Orchestrator Agent (per step LLM)  │
                              │   按方法论整合 → 最终回答            │
                              └──────────────────────────────────────┘
                                              ▲
                                              │ user feedback / teaching
                                              │
                                       ┌──────┴───────┐
                                       │ Skill Trainer│  ← 教导意图识别
                                       └──────────────┘

       横切关注点(BaseAgent / Trace / Logger / WebSocket Push):
       ┌──────────────────────────────────────────────────────────┐
       │   infra: LLMClient / Logger / Config                      │
       │   core : BaseAgent / Context / CapabilityAnalyzer / Plugin │
       └──────────────────────────────────────────────────────────┘
```
